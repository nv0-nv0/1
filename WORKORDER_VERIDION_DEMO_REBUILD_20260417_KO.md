# NV0 Veridion 데모/결제/발행 구조 재정의 작업지시서 및 처리결과

작업일: 2026-04-17
대상: Veridion URL 기반 준법·규제 리스크 데모, 결제 후 발행 구조

## 목표
- 데모에서 사이트 주소 입력만으로 법률·규제 위배 가능 항목을 실제로 보여주기
- 무료 단계에서 구매 판단이 가능한 핵심 수치만 노출하기
- 결제 후에는 2개의 납품형 서비스가 자동으로 발행되도록 구조를 명확화하기

## 요구 반영 사항
### 1) 데모에서 바로 보여주는 항목
- 사이트 주소 입력
- 영역별 위반 가능 항목 건수
- 위기 점수
- 유사 업종 대비 하위 퍼센트
- 예상 최대 과태료
- 추가 운영 지표
  - 정책 고지 완성도
  - 개인정보 안내 준비도
  - 결제·환불 고지 준비도
  - 소비자 분쟁 유발 가능성

### 2) 결제 후 발행 구조
- 서비스 1 · 전체 세부 점검 리포트
  - 전체 이슈 상세표
  - 영역별 집계
  - 페이지별 결과 및 조치
  - 예상 최대 과태료/비금전 리스크
- 서비스 2 · 사이트 맞춤형 수정안 리포트
  - 사이트 맞춤형 교체 문구
  - 개인정보/환불/약관/사업자 정보 수정안
  - 적용 위치 및 우선순위
  - 재점검 체크리스트

## 실제 처리 내용
### 백엔드
- `server_app.py`
  - Veridion 업종 비교 기준 추가
  - 유사 업종 하위 퍼센트 계산 로직 추가
  - 리스크 진단 패널(운영 지표) 추가
  - 결제 후 발행 서비스 번들(`serviceBundle`) 추가
  - 공개 리포트/데모 리포트에 서비스 1·2 노출 구조 반영
  - 결과 발행본(`resultPack`)에 서비스 1·2를 기본 번들로 포함

### 프런트엔드
- `src/assets/site.js`
  - Veridion 데모 결과 카드 재구성
  - KPI에 위기 점수/유사 업종 하위 퍼센트/예상 최대 과태료 반영
  - 영역별 위험 표, 추가 운영 지표, 결제 후 발행 서비스 표 추가
  - 결제 완료 후 안내 문구를 서비스 1·2 기준으로 재작성
  - 데모 입력폼 안내 문구 강화

- `src/data/site.json`
  - Veridion 헤드라인/요약/결과물/데모 시나리오/워크플로/가격 기준 업데이트
  - 플랜 포함 항목에 서비스 1·2 기준 반영
  - FAQ에 결제 후 차이점 추가

### 빌드
- `python build.py` 실행
- `dist/` 전체 재생성 완료

## 검증
- `python -m py_compile server_app.py` 통과
- 로컬 서버 기동 후 `/api/public/veridion/scan` 호출 성공
- 확인 항목
  - 위기 점수 반환 성공
  - 유사 업종 하위 퍼센트 반환 성공
  - 서비스 1/2 반환 성공

## 산출물 핵심 정의
### 무료 데모
- URL 기반 자동 점검
- 상위 이슈 공개
- 영역별 건수/위기 점수/유사 업종 하위 퍼센트/예상 최대 과태료 공개
- 결제 후 열리는 서비스 1·2 사전 고지

### 유료 발행본
- 서비스 1: 전체 세부 점검 리포트
- 서비스 2: 사이트 맞춤형 수정안 리포트

## 주의
- 과태료/상대 위치 수치는 자동 점검 기반 추정치이며 법률 확정 판단이 아님
- 실제 제재 여부는 관계 법령, 사실관계, 소관 기관 판단에 따라 달라질 수 있음
