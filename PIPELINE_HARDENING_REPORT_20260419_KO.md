# NV0 파이프라인 연계 강화 보고서 (2026-04-19)

## 이번 보강 목적
데모 → 결제 → 결제성공 → 진행정보 입력 → 고객포털 조회 흐름이 중간 단계에서 끊기지 않도록
프런트 상태 연계와 자동 프리필을 강화했다.

## 적용한 핵심 보강
1. 파이프라인 컨텍스트 저장소 추가
- `sessionStorage` + `localStorage`에 최근 데모/주문/포털 조회 상태를 동시 저장
- 키: `nv0-pipeline-context`

2. 데모 결과 자동 연계 강화
- Veridion 데모 성공 시 다음 정보 저장
  - 제품
  - 사이트 주소
  - 국가/중점 확인사항
  - 리포트 ID / 리포트 코드
  - 후속 결제용 note
- 이후 체크아웃 폼에서 자동 프리필 가능

3. 체크아웃 프리필 강화
- 제품 상세 결제폼 / 공용 체크아웃폼 모두 최근 컨텍스트 기반 자동 채움
- `reportId`, `reportCode`, `link`, `note` 연계
- 결제 직전에도 최근 제품/플랜/URL/이메일을 다시 저장

4. 결제 성공 페이지 연계 강화
- 결제 성공 시 최근 주문 상태를 저장
- intake 폼에서 회사명/담당자명/이메일/사이트주소를 자동 복원
- intake 저장 완료 후 최신 주문 상태를 다시 기록

5. 고객 포털 자동 연결 강화
- 포털 페이지 진입 시
  - URL query의 `email`, `code` 우선 사용
  - 없으면 최근 파이프라인 컨텍스트의 주문 정보 사용
- 값이 이미 있으면 자동 조회까지 수행
- 조회 성공 시 최신 주문 상태를 다시 저장

6. 문의 폼 연계 강화
- 최근 제품/회사명/담당자명/이메일/URL/메모를 문의 폼에도 자동 프리필

## 판단
이번 v3 보강본은 단순 페이지 복구 수준이 아니라
사용자 행동 흐름이 다음 단계로 자동 이어지도록 프런트 파이프라인을 강화한 버전이다.
특히 데모 결과와 결제 이후 포털 조회 사이의 연결 밀도를 높였다.
