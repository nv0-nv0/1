A. 작업 목표 재정의
최종 목표:
- NV0 공개/관리자 화면을 사용자 지시 기준으로 재정렬하고, Veridion 데모/결제/자료실 흐름을 실제 동작 기준으로 맞춘 뒤 납품 가능한 패키지로 재생성한다.
기대 인수 기준:
- 공개 네비게이션이 상단 중심으로 동작한다.
- 게시판 표기가 자료실로 바뀐다.
- Veridion 데모는 사이트 주소 입력 즉시 분석형으로 동작한다.
- 체크아웃은 제품/플랜 중심 최소 입력 후 외부 결제로 이어진다.
- 결제 후 진행 정보 입력으로 서비스 전달 상태까지 전환된다.
- 관리자에서 자료실 설정/직접 글쓰기/파일 업로드/전체 제품 CTA 발행을 처리할 수 있다.
범위:
- build.py
- server_app.py
- src/assets/site.js
- scripts/generate_compat_pages.py
- scripts/page_overrides.py
- dist 재생성 및 로컬 스모크 테스트
비범위:
- nv0.kr 실운영 서버 직접 배포
- 실운영 Toss 실결제 키 기준 검증
- Coolify/리버스프록시/SSL 최종 운영 점검
완료 기준:
- 핵심 페이지와 API 스모크 테스트 통과
- 최종 패키지 ZIP 생성
납품 판단 기준:
- 로컬 기준 즉시 검토/즉시 시연/인수 검토 가능
- 실운영 직접 배포는 별도
